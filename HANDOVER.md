## 5. API エンドポイント一覧

すべて `http://localhost:8000` 配下。

| メソッド | パス | 説明 | 主要パラメータ |
|----------|------|------|----------------|
| GET | `/api/metadata` | メタデータ取得 | — |
| GET | `/api/national` | 全国集計データ | `kpis`(カンマ区切り) |
| GET | `/api/by-type` | 設置形態別データ | `kpis`, `type` |
| GET | `/api/by-prefecture` | 都道府県別データ | `kpis`, `prefecture` |
| GET | `/api/universities` | 大学リスト（検索・ソート・ページング） | `type`, `prefecture`, `search`, `kpi`, `sort`, `limit`, `offset` |
| GET | `/api/university/:code` | 大学詳細 | `code`(パスパラメータ) |
| GET | `/api/ranking` | KPIランキング | `kpi`(必須), `year_index`, `type`, `prefecture`, `limit` |
| GET | `/health` | ヘルスチェック | — |

### レスポンス例

## 8. 既知の課題・注意点

1. **R2/R3（2020-2021）の学校コード欠損**: 大学名→コードのマッピングで解決済み。約60件は名前バリエーション（例: 東京藝術大学 vs 正規化名）により未解決で除外されている。

2. **様式6-1/6-2/7/9/10のKPI抽出**: 列名のパターンマッチでは誤列を取得する問題があったため、明示的な列インデックスによる抽出に切り替え済み（`extract_dashboard_data.py` 参照）。

3. **設置形態「不明」**: 学校コードから設置形態を判定できない一部の大学が「不明」に分類されている。

4. **フロントエンドはBabel Standaloneでブラウザトランスパイル**: 本番運用時はビルドステップ（Vite等）の導入を検討すべき。

5. **旧Python版のファイルが残っている**: `dashboard/backend/` に `main.py`, `data_loader.py` 等が残っているが、Node.js版（`server.js`）に完全移行済み。削除しても問題ない（`backend/data/dashboard_data.json` だけは必要）。

## 9. データ再抽出が必要な場合

JSONデータの再生成が必要な場合は、Coworkセッション内で作成した抽出スクリプトを使う:

```
# Coworkサンドボックス内のパス（ローカルにはコピーされていない可能性あり）
/sessions/serene-modest-rubin/extract_dashboard_data.py
```

このスクリプトは `openpyxl` で `産学連携_全大学_経年データ.xlsx` を読み込み、58 KPIを抽出して `dashboard_data.json` を出力する。

### 抽出スクリプトの主要ロジック

- **EXTRACT_DEFS辞書**: 様式2〜5の列名完全一致でKPIカラムを特定
- **EXTRACT_BY_INDEX辞書**: 様式6-1/7/9/10は列インデックス（0始まり）で特定
- **名寄せ処理**: R2/R3のNULL学校コードを大学名→コードマッピングで補完
- **集計**: 大学別 → 全国合計 / 設置形態別 / 都道府県別に3段階集計
